Introduction

Before you start, first make sure you’ve set up your github repo correctly following the instructions in the “Before You Begin” section above.

Today we are going to continue our journey of modelling the distribution of the Frilled Lizard (Chlamydosaurus kingii) with one of the most widely used tools in ecology: species distribution models (SDMs). Last week, we used exclusively geometric models which rely on presence-only data to model geographic range polygons. This week, we will use a class of SDMs that uses both presence and absence (or pseudo-absence) data to model suitable habitat based on environmental conditions.

These models underpin:

  • conservation planning
  • invasive species forecasts
  • climate change projections
  • biodiversity policy

They are powerful tools but very easy to misuse. Throughout today’s workshop, focus less on running models and more on understanding assumptions.

Extracting Environmental Values

To model a niche, we need predictor values at occurrence locations. First, however, we want to drop points which are duplicated in a grid cell. This removes some bias from having some grid cells with many observations (e.g., close to cities) and some with very few. All grid cells with a sighting will contribute just once to our models.

# prune data to one point per grid cell
# Create SpatVector
occ_v <- vect(occ, geom = c("X", "Y"), crs = crs(bioclim_data))

# Get raster cell IDs
occ_v$cell_id <- cellFromXY(bioclim_data, crds(occ_v))

# Keep one point per raster cell
occ_pruned <- occ_v[!duplicated(occ_v$cell_id), ]

# Extract climate
occ_env <- terra::extract(
  bioclim_data,
  occ_pruned,
  ID = FALSE
) |> 
  as_tibble()

occ_model <- bind_cols(
  as_tibble(crds(occ_pruned)) |> 
    setNames(c("longitude", "latitude")),
  occ_env
) |> 
  mutate(presence = 1)

Accessible Area Background

In this method, we sample points within a fixed distance of our occurrence points, using a buffer, just like in last weeks workshop. Here, we are assuming that the species could realistically disperse only within some distance of known occurrences. Further, we are also limiting the range of environments we will contrast with our occurrence points, which increase the specificity of the model to our species.

We buffer occurrence points to define an accessible area.

occ_proj <- st_transform(occ_sf, 3577)

Environmental Envelope Background

In this method, we sample points within Australia that are within the broad environmental range of our occurrence points. Here, we are assuming that the species could realistically disperse only within the climatic conditions that it could experience or have access to across its distribution. If its range is determined by climate, then it should only occupy a subset of the total range of conditions in the area. Further, we are also limiting the range of environments we will contrast with our occurrence points which should increase the specificity of the model to our species.

In other words, instead of restricting sampling background points in geographic space, we restrict environmental space.

We sample only within the environmental range + one standard deviation observed at presence locations.

env_ranges <- occ_env %>% 
  summarise(
    across(
      everything(),
      list(min = ~min(.x, na.rm = TRUE)-sd(.x, na.rm = TRUE),
           max = ~max(.x, na.rm = TRUE)+sd(.x, na.rm = TRUE))
    )
  )

clim_df <- terra::as.data.frame(
  bioclim_data,
  xy = TRUE,
  na.rm = TRUE
)

ranges_long <- env_ranges %>% 
  pivot_longer(
    everything(),
    names_to = c("variable", ".value"),
    names_sep = "_(?=[^_]+$)"   # splits on final underscore only
  )

filtered_cells <- clim_df %>% 
  
  pivot_longer(
    cols = -c(x, y),
    names_to = "variable",
    values_to = "value"
  ) %>% 
  
  left_join(ranges_long, by = "variable") %>% 
  
  filter(value >= min,
         value <= max) %>% 
  
  group_by(x, y) %>% 
  summarise(n = n(), .groups = "drop") %>% 
  
  filter(n == nrow(ranges_long)) %>% 
  
  select(x, y)

bg <- filtered_cells %>% 
  slice_sample(n = 5000)

bg_env <- terra::extract(bioclim_data, bg, ID = FALSE) %>% 
  as_tibble()

bg_model <- bind_cols(
  bg %>% 
    rename(longitude = x, latitude = y),
  bg_env
) %>% 
  mutate(presence = 0)

Bias Correction Background

In species distribution modeling, the choice of background points (locations representing where a species could potentially occur) can strongly influence the model. One way to make these background points more realistic is to account for sampling effort — that is, where people have actually looked for the species.

Here, we use a bias-correction approach based on the distribution of Agamid lizards in Australia:

  • The Frilled Lizard (Chlamydosaurus kingii) belongs to the family Agamidae, which includes over 70 species in Australia.

  • If a researcher is out recording Agamid lizards, it is very likely that they would record the Frilled Lizard if it were present.

  • Therefore, areas where Agamid lizards have been observed represent locations that have actually been sampled, and are reasonable places to draw background points.

  • By sampling background points from these areas, we avoid including locations where no one has ever looked, which could otherwise bias the model by suggesting that absence is equivalent to unsurveyed locations.

In other words, this method uses existing survey effort as a guide to produce more realistic background points, improving the reliability of our species distribution models.

# Read in distribution of all agamid species
agamid_occ <- read.csv("data/agamidae.csv")

# Convert to terra vector object
agamid_occ <- terra::vect(
  agamid_occ,
  geom = c("decimalLongitude", "decimalLatitude"),
  crs = "EPSG:4326"
)

# Rasterize occurrence counts onto first climate layer
# This gives a "density" surface of sampling effort
agamid_density <- terra::rasterize(
  x = agamid_occ,
  y = bioclim_data[[1]],
  fun = "sum"
)

# Sample random points proportional to density ("bias correction")
# Use stratified method to weight points by existing sampling effort
set.seed(123)  # for reproducibility
bck_bias <- terra::spatSample(
  x = bioclim_data[[1]],
  size = 5000,
  method = "stratified",
  weights = agamid_density,
  as.points = TRUE,
  na.rm = TRUE
)

# Extract coordinates for tidyverse manipulation
bg <- as_tibble(terra::crds(bck_bias)) %>% 
  rename(longitude = x, latitude = y)

# Extract environmental values for each point
bg_env <- terra::extract(bioclim_data, bg, ID = FALSE) %>% 
  as_tibble()

# Combine into modeling-ready dataframe
bg_model <- bind_cols(
  bg,
  bg_env
) %>% 
  mutate(presence = 0)

Fit Models

Now that we have presence and background data, along with the environmental variables extracted at each site, we can move on to the modeling part of the workshop.

First lets combine our occurrence and background data into a single data frame.

sdm_data <- bind_rows(occ_model, bg_model)

# Check the result
glimpse(sdm_data)

We will fit two types of models:

  • Generalized Linear Model (GLM): Models linear relationships between predictors and the response (presence/absence). Here we use logistic regression.
  • Generalized Additive Model (GAM): Can capture non-linear relationships using smoothing functions (s()), allowing the model to flexibly fit more complex patterns.

⚠️ Note: In real-world SDMs, models are typically fitted at high spatial resolution with many predictors and more advanced statistical or machine learning techniques. Here, we use a simplified workflow to focus on the fundamentals.

Task 3. Select predictors

Selecting environmental predictors is critical. Variables should be chosen based on ecological relevance and to avoid collinearity. Both GLMs and GAMs do poorly when predictors are highly correlated.

# Check for collinearity among predictors
library(caret)

correlation_matrix <- cor(sdm_data[, 3:21], use="complete.obs")
correlated <- findCorrelation(correlation_matrix)
names_correlated <- colnames(sdm_data[,3:21])[correlated]

# Remove highly correlated variables
sdm_data_uncor <-sdm_data[, !colnames(sdm_data) %in% names_correlated]

# Keep track of predictors
predictor_vars <- colnames(sdm_data_uncor)[grep("Bio", colnames(sdm_data_uncor))]
print(predictor_vars)

Next, we define formulas for GLM and GAM models. For the GAM, k = 4 sets the flexibility of the splines: increase k for more complex curves, decrease for simpler curves.

# GLM: linear model
formula_glm <- formula(paste0("presence ~ ", paste0(predictor_vars, collapse = " + ")))

# GAM: non-linear model with splines
formula_gam <- formula(paste0("presence ~ ", paste0("s(", predictor_vars,", 4)", collapse = " + ")))

# Inspect formulas
print(formula_glm)
print(formula_gam)

Task 4. Fit models

Now we fit both models using the uncorrelated predictors.

library(gam)

# Fit GLM (logistic regression)
glm_sdm <- glm(formula_glm, data = sdm_data_uncor, family = binomial(link = "logit"))
summary(glm_sdm)

# Fit GAM
gam_sdm <- gam::gam(formula_gam, data = sdm_data_uncor, family = binomial(link = "logit"))
summary(gam_sdm)

We can turn a probabilistic measure of habitat suitability into a binary species dsitribution by thresholding the predictions. To do this we need to evaluate our models and assess how well the model predicts our known occurence records.

A common approach is the Receiver Operating Characteristic (ROC) curve, which plots the trade-off between true positive rate and false positive rate across all possible thresholds.

From the ROC curve, we calculate the Area Under the Curve (AUC):

  • AUC ≈ 1 → the model perfectly separates presences from absences

  • AUC ≈ 0.5 → the model performs no better than random guessing


> ⚠️ Note: Higher AUC values indicate better predictive performance, but AUC alone does not capture all aspects of model reliability — it should be interpreted alongside other evaluation metrics.
library(dismo)

# Split presence and background
p <- sdm_data_uncor[sdm_data_uncor$presence==1, ]
a <- sdm_data_uncor[sdm_data_uncor$presence==0, ]

# Evaluate models
glm_eval <- evaluate(p=p, a=a, model=glm_sdm, type='response')
gam_eval <- evaluate(p=p, a=a, model=gam_sdm, type='response')

# Plot ROC curves
par(mfrow=c(1,2))
plot(glm_eval, "ROC")
plot(gam_eval, "ROC")

Take a look at the evaluation objects and try plotting some other summary statistics, like kappa, the true positive rate (TPR), false positive rate FPR, etc.

The aim is to find the value of habitat suitability, which has the maximum value of a metric like spec_sens - the threshold at which the sum of the sensitivity (true positive rate) and specificity (true negative rate) is highest. Then we say all sites with a habitat suitability higher than that threshold is classed as presence, and every cell with a habitat suitability lower than the threshold is classed as an absence.

pred_glm_threshold <- threshold(glm_eval, stat = "spec_sens")
pred_gam_threshold <- threshold(gam_eval, stat = "spec_sens")

pred_glm_bin <- pred_glm > pred_glm_threshold
pred_gam_bin <- pred_gam > pred_gam_threshold


glm_bin_plot <- leaflet() |>
  addProviderTiles(providers$CartoDB.Positron) |>

Model Validation

So far we have evaluated the model on the same data used to fit it, so it may appear to perform very well. This is not a realistic estimate of predictive performance, because in real-world applications we want the model to generalize to new locations.

A better way to estimate predictive performance is to split the data into training and testing subsets, fit the model on the training data, and evaluate it on the testing data.

Task 5. K-Fold Cross-Validation

K-fold cross-validation automates the validation process by splitting the data into k partitions and repeats the evaluation on each partition. As follows:

  1. Split the data into K folds

  2. Repeat K times:

    • Train the model on K – 1 folds

    • Test the model on the remaining fold

  3. Combine results across folds to get an average AUC and measure of variability

K <- 5
k_fold <- kfold(sdm_data_uncor, k = K)

# Set up table to store results
auc_df <- data.frame(k = 1:K, glm_AUC = NA, gam_AUC = NA)

for (k in 1:K) {
  
  # Split into training and testing data
  train <- sdm_data_uncor[k_fold != k, ]
  test  <- sdm_data_uncor[k_fold == k, ]
  
  # Refit models on training data
  mod_glm <- update(glm_sdm, data = train)
  mod_gam <- update(gam_sdm, data = train)
  
  # Evaluate on test fold
  glm_eval_tmp <- evaluate(
    p = test[test$presence == 1, ],
    a = test[test$presence == 0, ],
    model = mod_glm,
    type = 'response'
  )
  
  gam_eval_tmp <- evaluate(
    p = test[test$presence == 1, ],
    a = test[test$presence == 0, ],
    model = mod_gam,
    type = 'response'
  )
  
  # Save AUC for each fold
  auc_df[k, "glm_AUC"] <- glm_eval_tmp@auc
  auc_df[k, "gam_AUC"] <- gam_eval_tmp@auc
}

# Inspect AUC across folds
print(auc_df)

By repeating the evaluation on different subsets of the data, k-fold cross-validation provides a more robust estimate of model predictive performance than a single train/test split.

Model Predictions Through Time

If we’re happy with our model in the present-day and convinced we have accurately modelled the niche of the species, we might want to ask how the species distribution has changed through time.

Commonly, people look to model the change in distribution into the future using models of climate change under different emissions scenarios. Equally, we might also want to know how the species was distributed in the past to understand how population structure might have evolved over time when conditions were vastly different to today.

One such period is the Last Glacial Maximum ~21,000 years ago. During the LGM, Australia looked very different, it was a colder and drier place than the present day and many species may have contracted their geographic ranges to warmer and more mesic refugia.

Lets see what our SDM predicts for the distribution of the Frilled Lizard at the LGM!

First, lets load the LGM climate data make sure the names match the WorldClim data

# load bioclim data for the LGM (from CHELSA)
lgm_bioclim_data <- rast("data/chelsa_australia_LGM.tif")

# The names of the different variables must match exactly
names(bioclim_data)
names(lgm_bioclim_data)

(2) Cross-validation with spatial folds

In the workshop so far, we have used random folds in K-fold cross validation. These generally show consistent high performance across folds. However, to assess whether a model is transferable in space and time, it is better to split the data in a more structured way. For example, by splitting into spatial folds, we can ask whether the distribution of a species in one restricted part of its range can be informative about where it should be distributed in other parts of the continent. A robust model should be transferable. A rough script to create 6 spatial folds is below but check out the blockCV package for even more options to do this.

# Create 2D spatial blocks
sdm_data_uncor <- sdm_data_uncor %>%
  mutate(
    lon_bin = ntile(longitude, 3),
    lat_bin = ntile(latitude,  2),
    spatial_fold = as.numeric(interaction(lon_bin, lat_bin))
  )

# If number of folds exceeds K, randomly reduce to K
K <- length(unique(sdm_data_uncor$spatial_fold))

auc_df_spatial <- data.frame(k = 1:K, glm_AUC = NA, gam_AUC = NA)

for (k in 1:K) {
  
  # Spatial split
  train <- sdm_data_uncor[sdm_data_uncor$spatial_fold != k, ]
  test  <- sdm_data_uncor[sdm_data_uncor$spatial_fold == k, ]
  
    if (length(unique(test$presence)) < 2) {
    next
  }

  
  # Refit models
  mod_glm <- update(glm_sdm, data = train)
  mod_gam <- update(gam_sdm, data = train)
  
  # Evaluate
  glm_eval_tmp <- evaluate(
    p = test[test$presence == 1, ],
    a = test[test$presence == 0, ],
    model = mod_glm,
    type = "response"
  )
  
  gam_eval_tmp <- evaluate(
    p = test[test$presence == 1, ],
    a = test[test$presence == 0, ],
    model = mod_gam,
    type = "response"
  )
  
  auc_df_spatial[k, "glm_AUC"] <- glm_eval_tmp@auc
  auc_df_spatial[k, "gam_AUC"] <- gam_eval_tmp@auc
}
